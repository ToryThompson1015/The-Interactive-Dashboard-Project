The Interactive Dashboard Project

A collaborative activity to review and apply your React and JavaScript fundamentals.

This project will guide you through building a simple product catalog dashboard. The primary goal is to practice working as a team, managing a shared codebase with Git, and reinforcing core programming concepts. In this guide, you'll find a complete plan, from initial setup to final presentation, designed to help your team succeed.

3-4 Hours Duration
3 Learners per Group
2+ Core Skills (JS & React)

Project Requirements

  • Use a public API of your choice to fetch the product data.
  • Handle loading and error states for the API call.
  • Render the product list using the `.map()` method.
  • Build a reusable `ProductCard` component that accepts props.
  • Implement a search bar to filter products by name.
  • Use `useState` for product data and the search term.
  • **Style the application to be visually appealing and responsive.**
  • **Create a `README.md` file in your repository.** This file should contain a clear description of the project and a list of team members and their assigned roles.
  • **Include a link to your team's GitHub repository** in your final project submission.

Pro Tips for Teamwork

Avoid these common pitfalls to ensure your team collaboration is a success.

Pitfall #1: Merge Conflicts

This happens when two people change the same part of a file, and Git doesn't know which version to keep.

Solution: Always sync with the `main` branch before starting new work. Run `git pull origin main` often to stay updated and resolve smaller conflicts early.

Pitfall #2: Communication Breakdown

Duplicate work or confusion arises when team members aren't aware of what others are doing.

Solution: Start with a quick 5-minute meeting to assign tasks. Use GitHub Issues or a simple to-do list to keep track of who is working on what.

Pitfall #3: "Cowboy Coding"

Pushing code directly to the `main` branch without review can introduce bugs and break the application for everyone.

Solution: Strictly follow the workflow. Feature branches and Pull Requests are mandatory to ensure all code is reviewed before being merged.

Pitfall #4: Uneven Workload

It's easy for one person to end up doing most of the work if tasks aren't clearly defined and distributed.

Solution: The Project Lead should break the project into small, specific tasks and assign them clearly. Regular check-ins help ensure everyone is contributing.
